## unifi-mgr v0.1.7 — pre-cron safety release

Патч поверх 0.1.6. Закрывает блокеры, найденные аудитом 2026-06-11, перед включением Phase 5 cron: утечку секретов устройств в лог, краши cron-алертинга, гонку cooldown-истории, двойные рестарты и недетерминированный деплой. Несёт всё из 0.1.6.

### Безопасность

- **`audit full --json` больше не сливает секреты устройств в лог.** Команда отдавала сырой `/stat/device` payload (`x_authkey`, `x_aes_gcm`, `x_ssh_hostkey_fingerprint`, …) — а Stage 1 runbook пишет её вывод в cron-лог. Теперь по умолчанию применяется общий рекурсивный санитайзер (`utils/redact`); сырой дамп — только явный `--raw` с предупреждением в stderr.
- **Тот же санитайзер чинит вложенную утечку в `export devices`.** Allow-list резал только top-level колонки; секреты, вложенные в `uplink` (модель с `extra="allow"`), протекали. Теперь вырезаются рекурсивно.
- **Telegram bot-token больше не утекает в лог.** При неудачной отправке текст исключения содержал URL `…/bot<TOKEN>/sendMessage`; токен вырезается перед логированием.

### Надёжность / корректность

- **`audit critical` / `audit light` не падают на одном битом payload.** Один device/client без обязательных полей ронял всю cron-проверку (а с ней — весь алертинг). Теперь битая запись пропускается с warning, остальные обрабатываются.
- **Cooldown-история перечитывается под локом (`auto`/`profile`/`device`).** История читалась в конструкторе до захвата FileLock — параллельный ран мог затереть свежие записи stale-снапшотом (lost-update) и разрешить повторный рестарт. Теперь перечитывается под локом.
- **Команды рестарта не ретраятся.** `send_command` (`/cmd/devmgr`) — неидемпотентный POST; повтор после ambiguous timeout/5xx рестартовал AP или power-cycle'ил порт дважды. `retries=1` это исключает (GET-запросы ретраятся как раньше).
- **Telegram rate-limit тратится только при успешной отправке.** Раньше бюджет списывался до HTTP-запроса — провал блокировал немедленный retry того же алерта. Теперь порядок `check → send → record`.

### Зависимости / деплой

- **Верхние границы по мажорам зависимостей.** `pydantic<3`, `typer<1`, `pydantic-settings<3` и др. — deploy резолвит deps из PyPI, и без cap следующий мажор молча ломал бы прод.
- **`constraints.txt` + `deploy.sh -c`.** Прямые runtime-зависимости пиннятся к проверенным версиям; `deploy.sh` подхватывает файл из бандла автоматически. Воспроизводимость прод-установок.

### Развёртывание

Артефакты: `unifi-mgr-0.1.7-deploy.tar.gz` / `.zip`. 354 теста, ruff + mypy --strict чисто, бандл LF-clean с perms/CRLF/notes build-gate. Деплой-пайплайн (install → deploy → verify) провалидирован end-to-end в Docker (Debian/py3.12), пины `constraints.txt` подтверждены на установке.
